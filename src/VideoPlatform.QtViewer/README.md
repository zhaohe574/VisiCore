# Windows 原生查看客户端

正式查看端采用 Qt 6 Widgets、C++20 与 libmpv，不包含 WebView 或浏览器内核。客户端只连接中心 API 和授权视频网关，不保存设备地址、设备账号或设备密码。

## 构建依赖

- Qt 6.5 或更高版本，需包含 Core、Network、Widgets、OpenGLWidgets 与 Svg。当前已验证 Qt 6.12.0。
- CMake 3.24 或更高版本。
- Ninja 与 Qt 预编译包匹配的 64 位工具链。当前已验证 Qt 6.12.0 与 MinGW 13.1.0 x64。
- libmpv x64 运行时 `mpv-2.dll` 或 `libmpv-2.dll`。客户端运行时解析稳定 C API，不要求导入库。

界面依赖固定为 Qlementine 1.4.2、Qt ADS 5.0.0 与 Lucide 1.23.0。CMake 会核验官方归档 SHA-256：Qlementine 静态链接进主程序，Qt ADS 构建为独立动态库，Lucide 只把查看端使用的 SVG 编入 `:/icons/lucide/` 资源前缀。普通开发配置可从固定官方 HTTPS 地址获取归档；受控或离线构建可分别设置 `VIDEOPLATFORM_QLEMENTINE_SOURCE_ARCHIVE`、`VIDEOPLATFORM_QT_ADS_SOURCE_ARCHIVE`、`VIDEOPLATFORM_LUCIDE_SOURCE_ARCHIVE`，路径所指归档必须与固定哈希一致。

设置 `QT_ROOT` 后执行；安装阶段可选设置 `MPV_ROOT`。安装器会从其根目录或 `bin` 子目录复制 `mpv-2.dll` 或 `libmpv-2.dll`：

```powershell
cmake --preset windows-release
cmake --build --preset windows-release
cmake --install .\build\release
```

离线构建示例：

```powershell
cmake --preset windows-release `
  -DVIDEOPLATFORM_QLEMENTINE_SOURCE_ARCHIVE='D:\approved-source\qlementine-v1.4.2.tar.gz' `
  -DVIDEOPLATFORM_QT_ADS_SOURCE_ARCHIVE='D:\approved-source\qt-ads-5.0.0.tar.gz' `
  -DVIDEOPLATFORM_LUCIDE_SOURCE_ARCHIVE='D:\approved-source\lucide-1.23.0.tar.gz'
```

资源接口保留 `unlock.svg`、`more-horizontal.svg` 与 `circle-help.svg` 三个兼容名称；它们在 Lucide 1.23.0 中分别映射到 `lock-open.svg`、`ellipsis.svg` 与 `circle-question-mark.svg`。业务代码无需感知上游改名。

安装阶段会调用 Qt 部署工具复制所需 DLL 与平台插件，并单独安装 Qt ADS DLL。`mpv-2.dll` 或 `libmpv-2.dll` 必须来自可信 x64 发行包，并通过文件哈希纳入发布清单。正式 Windows 发行应在独立发行流程中固定运行时版本、来源、许可证文本、第三方许可材料和代码签名状态；不得直接分发未经验证的构建目录。

安装验收可使用以下命令检查动态运行时是否可被加载、所需 C API 是否齐全，以及 API 主版本是否兼容。该检查不登录中心、不申请视频会话，也不访问录像机或摄像头：

```powershell
.\VideoPlatformQtViewer.exe --mpv-library 'C:\Program Files\VideoPlatform\mpv-2.dll' --verify-mpv-runtime
```

`--mpv-library` 只接受本机绝对路径，适用于安装验收和受控的运行时覆盖；常规生产发布仍应将审核通过的 `mpv-2.dll` 与客户端放在同一目录。用于验收的运行时文件也必须记录版本、来源和 SHA-256，不得将临时下载路径写入安装配置。

界面视觉验收可使用独立的空数据工作台，不登录、不访问外部服务，也不会读写正式账号的停靠布局：

```powershell
.\VideoPlatformQtViewer.exe --verify-ui-shell --verify-ui-size 1366x768
```

`--verify-ui-size` 只能与 `--verify-ui-shell` 一起使用，范围为 `1080x680` 至 `7680x4320`。可分别传入 `1366x768`、`1920x1080` 和 `2560x1440`，检查标题栏命中区域、工具栏溢出菜单、停靠面板宽度和文字裁切；DPI 缩放仍需在对应 Windows 显示设置下执行。

若要覆盖一次完整的播放器初始化和嵌入播放，可提供一段本地测试媒体。该命令不会登录中心或读取摄像头：

```powershell
.\VideoPlatformQtViewer.exe --mpv-library 'C:\Program Files\VideoPlatform\mpv-2.dll' --verify-mpv-playback 'C:\VideoPlatform\verification.mp4'
```

离线播放校验在 15 秒内收到媒体开始事件即通过；超时或播放器错误将返回非零退出码。发布验收媒体应为受控的无敏感内容短片，不应使用真实监控录像。

需要校验网络 HLS 时，可使用受控网关的 HTTPS 地址；HTTP 仅允许本机回环验收地址，且不接受 URL 用户信息。该命令不登录中心，适用于已由外部流程签发并限制有效期的测试播放地址：

```powershell
.\VideoPlatformQtViewer.exe --mpv-library 'C:\Program Files\VideoPlatform\mpv-2.dll' --verify-mpv-stream 'https://gateway.example/hls/session/ticket/playback/session/index.m3u8'
```

网络流校验在 15 秒内收到媒体开始事件即通过。生产验收应使用短期测试票据，命令行和日志中不得包含设备账号、设备密码或长期令牌。

## 64 分屏容量验收

`--verify-mpv-grid-manifest` 会在同一 Qt 原生窗口中并发创建 1 至 64 个 libmpv 播放器。清单必须是本机绝对路径的 JSON 文件，`streams` 中每项都是不同的短期受控 HLS 地址；不允许 URL 用户信息、重复地址、非 HTTPS 地址或非回环 HTTP 地址：

```json
{
  "streams": [
    "https://gateway.example/hls/会话一/index.m3u8",
    "https://gateway.example/hls/会话二/index.m3u8"
  ]
}
```

生产 64 分屏验收应由中心为 64 个不同摄像头签发短期子码流票据，再在目标终端执行：

```powershell
.\VideoPlatformQtViewer.exe `
  --verify-mpv-grid-manifest 'C:\VideoPlatform\capacity-64.json' `
  --verify-mpv-grid-duration-seconds 1800
```

全部播放器完成加载并持续到指定秒数后，命令输出不含流地址的 JSON，包含播放器数量、首路就绪、全量就绪和总耗时。持续时间范围为 1 至 3600 秒；任一路加载失败、60 秒启动超时或清单不合规都会返回非零退出码。此模式不登录中心、不申请会话，也不打印清单 URL；它只消费由外部受控流程预先签发的短期地址。通过 64 路模式不自动代表终端达标，仍须记录 CPU、GPU、显存、网卡、首帧、丢帧与持续时段指标。

当前实现覆盖内存登录会话、认证代次隔离、权限过滤目录、搜索、1／4／9／16／25／36／64 分屏、主／子码流策略、窗格会话重复请求去重、自动续租、单窗撤销、libmpv 嵌入播放与事件轮询、播放器实例与播放代次隔离、隐藏窗格解码资源释放，以及媒体短断后最多六次退避换票重连；超过上限才撤销会话。播放器固定嵌入 Windows 原生窗口，但不强制单一 D3D11／硬解参数，由受控的 libmpv 运行时按硬件能力协商输出和解码路径。独立分屏容量验收模式强制使用不同短期 HLS 地址并输出无敏感 URL 的就绪统计。退出时只发起一次中心注销并等待网络排空，登出后的旧响应不会污染新状态。生产环境仍需在目标 GPU 和 64 分屏容量矩阵中完成硬件能力分级验收。

## 现代监控双工作区

主窗口以“实时预览”和“录像回放”组成双工作区，两个工作区共用账号、资源标签、搜索框和授权设备树，但分别维护窗格映射、播放器、控制状态和流会话。顶部显示中心认证状态，底部状态栏显示当前反馈和客户端本地时间。切换工作区会释放离开页面的全部流会话和解码资源，同时保留摄像头分配、分屏规格、回放检索条件和本地操作偏好；再次进入时只恢复当前可见窗格，避免预览与回放在后台同时占用录像机和网关并发。

客户端按账号保存窗口大小、侧栏宽度、资源标签、预览与回放分屏、码流策略、轮巡来源和间隔、PTZ 面板可见性以及回放同步开关。缩小分屏时，超出当前可见范围的窗格只会停止解码和撤销会话，摄像头 UUID 映射不会被清空；重新扩大分屏后会按原窗格恢复。这种无损分屏与“停止并清空全部”是两种独立操作，只有显式清空才删除窗格映射。

### 资源树、拖放与批量开播

左侧资源区提供“监控点”“收藏”“我的视图”三个标签。摄像头按区域组织，并以状态图标区分状态未知、在线、疑似离线、离线和恢复中，悬停工具提示会显示对应中文状态与权限能力；目录搜索支持区域、摄像头编号和别名，刷新只重新读取当前账号的授权资源。

激活单个摄像头会在当前选中窗格打开；激活区域或选择“打开此区域全部监控点”会递归收集该区域及子区域中的摄像头，从当前窗格开始顺序分配，并自动选择能够容纳目标数量的最小分屏。预览最多分配 64 路，回放最多分配 4 路；权限不足或超过分屏容量的资源不会启动，并通过中文结果提示说明未分配数量。

资源树支持将摄像头或整个区域拖入预览、回放窗格。拖放 MIME 数据只携带去重后的稳定摄像头 UUID，不携带树行号、流地址、播放票据或设备凭据；放下时会重新在最新授权目录中解析 UUID，并在回放工作区过滤没有回放权限的摄像头。因此目录刷新、过滤、区域展开或排序不会使拖放目标错位，过期或越权 UUID 也不会被启动。

收藏和“我的视图”均按登录账号本地隔离。保存视图会记录当前预览分屏、全局码流策略以及每个已分配窗格的摄像头 UUID 和主／子码流选择，最多保存 20 个；应用视图时切回实时预览并仅恢复仍在当前授权目录中的摄像头。视图不保存流地址、会话票据、访问令牌或设备凭据。

### 实时预览

实时预览支持 1／4／9／16／25／36／64 分屏。全局码流策略可选“自动码流”“主码流”或“子码流”；自动模式在九分屏及以上使用子码流，单个窗格还可通过右键菜单临时切换主／子码流。鼠标移入窗格时显示摄像头名称、连接状态和码流类型，移出后自动隐藏；媒体短断时仍持续显示中文重连进度。双击窗格可单窗放大，再次双击或按 `Esc` 恢复；右键菜单提供重新连接、码流切换、最近 30 秒／1／3／5／10 分钟即时回放，以及停止并清空当前预览。

轮巡可选择收藏、全部在线摄像头或当前区域作为来源，按 5 至 300 秒间隔填充当前可见窗格。手动分配摄像头、资源失去授权、来源为空、切换工作区或退出客户端时会停止轮巡，轮巡不会保存或复用旧流地址和会话票据。

预览右侧 PTZ 面板覆盖八方向移动、立即停止、变倍、焦距、光圈以及 1 至 7 级速度。控件只在当前摄像头在线、设备声明 PTZ 能力且账号具备 PTZ 权限时启用，并直接显示不支持、无权限或离线等禁用原因。同一摄像头的 PTZ 命令严格串行并使用中心确认后的序号；按住控制键时每 1.2 秒续发短脉冲，松开、切换窗格、切换工作区或退出时立即排队 STOP。

### 录像回放

录像回放默认单窗，可切换为四窗。检索条件使用客户端本地日期和开始／结束时间，提供“今天”“昨天”和“近一小时”快捷范围，提交中心时统一换算为 UTC。每个窗格独立维护检索代次和“未检索／检索中／已加载／无录像／失败”状态，过期响应不会覆盖新条件；日历会标记当前检索结果覆盖的日期。

播放游标由 libmpv 的实际媒体位置驱动，并在 `paused-for-cache` 缓冲状态下冻结；控制完成后以 Worker 返回的位置重新建立媒体时间原点。中心 transport 轮询只刷新暂停状态、倍速和可用控制能力，不会使用静态位置反复覆盖正在播放的时间轴。

四窗默认加入同步组。启用“同步组”后，暂停、继续、定位和 0.5／1／2／4 倍速会并行下发到所有已加入同步组的可见窗格；通过窗格右键菜单取消“加入同步组”可使该窗格独立，选中独立窗格时控制只作用于自身。界面会根据所有目标会话共同声明的暂停、定位和变速能力启用控件，并聚合同步批次的成功与失败结果；设备未验收、并发受限、能力不支持和控制失败均显示中文说明，不展示内部错误代号。

底部多轨时间轴按可见回放窗格分别绘制真实录像片段和检索状态，普通录像、事件录像、近似覆盖和锁定录像使用不同颜色。点击轨道会同步选中对应窗格，并把定位位置吸附到该轨道最近的真实录像片段；空白区域不会盲目发送定位命令。时间轴支持 1 至 64 倍缩放、适应完整范围、滚轮围绕鼠标位置缩放、`Shift`＋滚轮平移，以及键盘左右键在当前轨道录像片段中定位。

ONVIF Profile G 回放保持同一发布路径，通过受控中继重启提交 `Range` 和读取速率；未声明、未验证或被设备拒绝的能力保持禁用。ONVIF 协议模拟器已覆盖定位、变速、幂等和能力降级，目标真机仍须按型号完成兼容性验收。本期不包含录像下载、裁剪、书签、截图和导出中心。

## 快捷键与窗格操作

- `Ctrl+F`：聚焦设备资源搜索框。
- `F11`：进入或退出应用全屏。
- `Esc`：优先退出单窗放大，其次退出应用全屏。
- `Delete`：清空当前选中窗格；焦点位于文本或时间输入框时不会触发。
- `Space`：在录像回放工作区暂停或继续当前控制目标；焦点位于输入控件时不会触发。
- 双击窗格：在当前工作区切换单窗放大与原分屏。
- 时间轴 `+`／`=`、`-`、`Home`：放大、缩小或恢复完整检索范围。
- 时间轴 `←`／`→`：沿当前选中轨道移动到邻近的可用录像位置。
- 时间轴滚轮：围绕指针缩放；`Shift`＋滚轮在已放大的范围内平移。

## 账号安全

侧栏账号菜单提供修改密码和退出登录。退出登录会立即停止轮巡与 PTZ 操作、释放全部窗格播放资源、清除内存中的访问令牌，并在中心注销完成或最多等待 2 秒后返回登录窗口；注销请求始终使用退出前的令牌。关闭主窗口仍会等待中心注销请求排空，最长等待 2 秒后退出程序。

登录窗口可选择记住账号，仅在登录成功后持久化用户名；取消勾选并成功登录会清除已记住的账号。客户端不会保存密码、访问令牌或设备凭据。

修改密码必须输入当前密码，新密码长度为 12 至 256 位，并且两次输入一致且不能与当前密码相同。客户端通过 `PUT /api/v1/auth/password` 提交当前密码和新密码；修改成功后中心撤销该账号的全部登录会话，客户端同步清除本地认证状态、释放播放资源并要求使用新密码重新登录。客户端不会持久化账号密码、访问令牌或修改密码请求内容。

中心管理平台新增账号时无需设置密码，初始密码与账号相同；管理员执行密码重置后，密码也会恢复为账号。首次登录或密码重置后的登录会在进入主界面前强制打开改密窗口，未完成改密只能返回登录界面，中心 API 同时拒绝该会话访问其他业务接口。

如果账号在查看端已经进入主界面后才被标记为必须改密，查看端识别中心返回的 `password_change_required` 错误码后也会立即弹出同一个强制改密窗口，不再只在底部状态栏显示错误。修改成功或返回登录都会退出当前工作台，防止继续使用受限会话。
