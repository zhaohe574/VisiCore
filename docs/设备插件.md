# 设备插件说明

## 内置运行时

开源核心仅提供 `onvif` 与 `direct-rtsp` 两种内置运行时。它们不依赖厂商 SDK，设备地址与凭据通过边缘节点的受控配置处理。

## 外部插件 manifest

外部插件必须声明唯一 `key`、版本、最低平台版本、协议、设备类型、端点、能力、配置 schema 与软件包信息。软件包信息必须包含镜像引用、不可变 `sha256:` 镜像摘要、包 SHA-256、签名公钥 ID 与 RSA SHA-256 签名。

平台只保存并验证 manifest，不拉取或执行镜像。部署者负责使用独立 Compose 覆盖文件启动插件容器，并在平台配置中提供受信任公钥。插件以 `external-edge` 运行时接收通用的录像检索、回放中继与 PTZ 命令。

## 安全要求

- 插件镜像必须固定到摘要，禁止使用可变标签作为信任依据。
- 插件容器不得挂载 Docker Socket、宿主根目录或核心服务数据卷。
- 厂商 SDK、许可证和设备凭据由部署者自行取得、只读挂载并保存在受控环境中。
- 插件升级必须保持适配器标识、运行时和既有设备端点兼容；正在使用的插件不能停用或卸载。
